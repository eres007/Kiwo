HOW IT WORKS
═══════════════════════════════════════════════════════════════════════════════

STEP 1: User Sets Up IDE
────────────────────────
1. User adds Kiwo MCP server to IDE config
2. Provides KIWO_TOKEN and KIWO_API_URL
3. Restarts IDE
4. Kiwo tools available in IDE

STEP 2: User Captures Memory
─────────────────────────────
1. User tells IDE: "Remember that we use Supabase"
2. IDE calls: capture_memory tool
3. Sends to backend: POST /api/memory/capture
4. Backend refines with AI
5. Memory stored in database

STEP 3: User Switches IDE
──────────────────────────
1. User opens different IDE (e.g., Claude Code)
2. User asks: "What database do we use?"
3. IDE calls: get_memories tool
4. Sends to backend: GET /api/memory/retrieve
5. Backend returns relevant memories
6. IDE shows: "We use Supabase"

STEP 4: Memory Synced Across All IDEs
──────────────────────────────────────
1. All IDEs connected to same Kiwo account
2. All memories stored in same database
3. Any IDE can access any memory
4. Perfect sync across all tools

═══════════════════════════════════════════════════════════════════════════════
MCP TOOLS AVAILABLE
═══════════════════════════════════════════════════════════════════════════════

1. capture_memory
   Input: content, category, tags
   Output: memory_id, success message
   Use: Save a new memory

2. get_memories
   Input: query, category (optional), limit
   Output: List of relevant memories
   Use: Retrieve memories for current task

3. get_project_context
   Input: project_id (optional)
   Output: All memories grouped by category
   Use: Get complete project overview

4. list_memories
   Input: page, limit
   Output: Paginated list of all memories
   Use: Browse all memories

5. delete_memory
   Input: memory_id
   Output: Success message
   Use: Delete a memory

═══════════════════════════════════════════════════════════════════════════════
API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

POST /api/memory/capture
├─ Input: { content, category, tags, source }
├─ Output: { success, memory_id, memory }
└─ Use: Capture new memory

GET /api/memory/retrieve?query=...&category=...&limit=5
├─ Input: query (required), category, limit
├─ Output: { success, memories, count }
└─ Use: Get relevant memories

GET /api/memory/list?page=1&limit=10
├─ Input: page, limit
├─ Output: { success, memories, total, pages }
└─ Use: List all memories with pagination

GET /api/memory/search?q=...&category=...
├─ Input: q (required), category
├─ Output: { success, results, count }
└─ Use: Search memories

PUT /api/memory/:id
├─ Input: { content, category, tags }
├─ Output: { success, memory }
└─ Use: Update memory

DELETE /api/memory/:id
├─ Input: memory_id
├─ Output: { success, message }
└─ Use: Delete memory

GET /api/memory/project-context
├─ Input: None
├─ Output: { success, context, total_memories }
└─ Use: Get full project context
