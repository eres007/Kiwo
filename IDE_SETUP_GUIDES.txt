═══════════════════════════════════════════════════════════════════════════════
STEP 0: GET YOUR KIWO TOKEN
═════════════════════════════

1. Sign up at https://kiwo.vercel.app
2. Log in
3. Go to Settings → API Tokens
4. Copy your token (starts with "kiwo_")
5. Keep it safe!

═══════════════════════════════════════════════════════════════════════════════
SETUP 1: CURSOR
═══════════════════════════════════════════════════════════════════════════════

Option A: Using MCP (Recommended)
──────────────────────────────────

1. Open Cursor Settings
   - Mac: Cmd + ,
   - Windows: Ctrl + ,

2. Go to: Features → MCP

3. Click "Add MCP Server"

4. Fill in:
   Name: Kiwo
   Type: Command
   Command: npx kiwo-mcp-server
   Environment Variables:
   - KIWO_TOKEN=your_token_here
   - KIWO_API_URL=https://kiwo.onrender.com

5. Click "Save"

6. Restart Cursor

7. You'll see Kiwo tools in the MCP panel

Option B: Using File Sync
─────────────────────────

1. Create .cursorrules file in your project root

2. Add:
   [Kiwo Memory Context]
   This project uses Kiwo for memory management.
   All project context is stored in Kiwo.

3. Kiwo will auto-update this file with your memories

═══════════════════════════════════════════════════════════════════════════════
SETUP 2: CLAUDE CODE (Terminal)
═════════════════════════════════

Claude Code is terminal-based, so use MCP:

1. Create ~/.claude/mcp.json (or ~/.claude/config.json)

2. Add:
   {
     "mcpServers": {
       "kiwo": {
         "command": "npx",
         "args": ["kiwo-mcp-server"],
         "env": {
           "KIWO_TOKEN": "your_token_here",
           "KIWO_API_URL": "https://kiwo.onrender.com"
         }
       }
     }
   }

3. Restart Claude Code

4. Use Kiwo commands:
   @kiwo capture_memory "We use Supabase for database"
   @kiwo get_memories "database decisions"
   @kiwo get_project_context

   - Neovim: Use packer.nvim or lazy.nvim

2. Add to your config:

3. Use commands:
   :KiwoCapture "Memory content"
   :KiwoSearch "query"
   :KiwoContext

HOW TO USE KIWO IN YOUR IDE
═══════════════════════════════════════════════════════════════════════════════

After setup, you can:

1. CAPTURE MEMORY
   Tell your IDE: "Remember that we use Supabase for database"
   IDE captures and sends to Kiwo
   Memory stored automatically

2. RETRIEVE MEMORY
   Ask your IDE: "What database do we use?"
   IDE retrieves from Kiwo
   Shows relevant memories

3. GET PROJECT CONTEXT
   Ask your IDE: "Show me project context"
   IDE retrieves all memories
   Shows complete project overview

4. SEARCH MEMORIES
   Ask your IDE: "Search for architecture decisions"
   IDE searches Kiwo
   Shows matching memories

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Q: MCP server not connecting?
A: 1. Check KIWO_TOKEN is correct
   2. Check KIWO_API_URL is correct
   3. Restart IDE
   4. Check IDE logs for errors

Q: Memories not being captured?
A: 1. Verify you're logged in
   2. Check token is valid
   3. Check API is running (https://kiwo.onrender.com/health)
   4. Check IDE console for errors

Q: Can't find Kiwo tools in IDE?
A: 1. Verify MCP server is added correctly
   2. Restart IDE
   3. Check IDE MCP settings
   4. Try reinstalling MCP server

Q: Token expired?
A: 1. Go to https://kiwo.vercel.app
   2. Log in
   3. Get new token from Settings
   4. Update IDE configuration
   5. Restart IDE
