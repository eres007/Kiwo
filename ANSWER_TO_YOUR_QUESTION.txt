PART 2: HOW CAN USERS USE IT FROM THE WEBSITE?
────────────────────────────────────────────────

STEP-BY-STEP USER FLOW:

STEP 1: USER VISITS WEBSITE
─────────────────────────────
1. User goes to: https://kiwo.vercel.app (after deployment)
2. Sees beautiful landing page with:
   - Header with navigation
   - Hero section explaining the problem and solution
   - Features section showing benefits
   - Pricing section with subscription tiers
   - FAQ section answering questions
   - CTA section encouraging signup
   - Footer with links

STEP 2: USER SIGNS UP
──────────────────────
1. User sees Hero section with email signup form
2. User enters their email address
3. User clicks "Start Free" button
4. Form validates email
5. Website sends: POST /api/auth/signup to backend
6. Backend creates user account in Supabase
7. Backend returns success response
8. Website shows: "✓ Check your email to get started!"

STEP 3: USER RECEIVES EMAIL
─────────────────────────────
1. User checks email inbox
2. Receives welcome email from Kiwo
3. Email contains:
   - Welcome message
   - Login link or credentials
   - Getting started guide
   - Next steps

STEP 4: USER LOGS IN
─────────────────────
1. User clicks login link in email
2. Or visits login page
3. Enters email and password
4. Submits login form
5. Website sends: POST /api/auth/login to backend
6. Backend validates credentials
7. Backend returns JWT token
8. Website stores JWT token
9. User is now authenticated

STEP 5: USER ACCESSES DASHBOARD
────────────────────────────────
1. User logs in successfully
2. Redirected to dashboard
3. Dashboard shows:
   - Welcome message
   - Setup wizard
   - Connected tools status
   - Memory statistics
   - Quick start guide

STEP 6: USER SETS UP PROJECT
──────────────────────────────
1. User creates project profile:
   - Project name
   - Tech stack
   - Project description
   - Team members (optional)

2. User connects AI tools:
   - Option 1: MCP Protocol (Cursor, Claude Code, etc.)
   - Option 2: File Sync (.cursorrules, CLAUDE.md)
   - Option 3: Chrome Extension (ChatGPT, Gemini, etc.)

3. User sets preferences:
   - Coding style
   - Communication preferences
   - Privacy settings
   - Notification settings

STEP 7: USER STARTS USING AI TOOLS
────────────────────────────────────
1. User opens Cursor IDE
2. Cursor has MCP connection to Kiwo
3. User starts coding on their project
4. User tells Cursor about their project:
   "We decided to use Supabase instead of Firebase because we need
    more control over the database and want to avoid vendor lock-in"

5. Cursor captures this conversation
6. When session ends, Cursor sends to Kiwo:
   POST /api/memory/capture
   {
     "raw_text": "We decided to use Supabase instead of Firebase...",
     "source": "cursor_session",
     "project_id": "proj_123"
   }

STEP 8: KIWO PROCESSES MEMORY
───────────────────────────────
1. Backend receives raw memory
2. Sends to Qwen3 AI model for refinement
3. AI extracts structured information:
   {
     "category": "architecture",
     "content": "Using Supabase instead of Firebase for database",
     "reason": "More control, avoid vendor lock-in",
     "importance_score": 0.95,
     "tags": ["database", "architecture", "decision"]
   }

4. Backend stores in database:
   - Structured JSON
   - Vector embedding (for semantic search)
   - Metadata (timestamps, importance, etc.)
   - Encrypted at rest

STEP 9: USER OPENS DIFFERENT AI TOOL
──────────────────────────────────────
1. User opens Claude Code IDE next day
2. Claude Code has MCP connection to Kiwo
3. Claude Code requests memories:
   GET /api/memory/retrieve?context=new_session

4. Backend retrieves relevant memories:
   - Runs semantic search
   - Filters by importance score
   - Returns top memories

5. Claude Code receives memories
6. Injects into system context (user doesn't see this)

STEP 10: AI TOOL HAS FULL CONTEXT
──────────────────────────────────
1. User asks Claude Code:
   "Should I use Firebase or Supabase for this new feature?"

2. Claude Code responds:
   "Based on your project context, you're already using Supabase
    and decided against Firebase due to needing more control and
    avoiding vendor lock-in. I'd recommend continuing with Supabase
    for consistency."

3. Developer never told Claude Code about Supabase
4. Developer never told Claude Code about the Firebase decision
5. Developer told Cursor yesterday
6. Claude Code knows it today
7. This is the magic moment - the core value of Kiwo

PART 6: HOW TO DEPLOY WEBSITE (5 MINUTES)
───────────────────────────────────────────

1. Go to https://vercel.com
2. Sign in with GitHub
3. Click "Add New..." → "Project"
4. Select repository: eres007/Kiwo
5. Configure:
   - Root Directory: ./website
   - Environment Variable:
     * Name: NEXT_PUBLIC_API_URL
     * Value: https://kiwo.onrender.com
6. Click "Deploy"
7. Wait 2-3 minutes
8. Website live at: https://kiwo.vercel.app
