1. WEBSITE (Frontend)
─────────────────────
Technology: Next.js + React + Tailwind CSS
Hosting: Vercel
URL: https://kiwo.vercel.app

Components:
- Header: Navigation, mobile menu
- Hero: Value proposition, email signup
- Features: Product benefits
- Pricing: Subscription tiers
- FAQ: Common questions
- CTA: Call to action
- Footer: Links and info

Features:
- Responsive design (mobile, tablet, desktop)
- Email signup form
- API integration with backend
- Environment variable configuration
- Error handling
- Loading states

2. BACKEND API
──────────────
Technology: Node.js + Express.js
Hosting: Render
URL: https://kiwo.onrender.com

Routes:
- /api/auth/* - Authentication endpoints
- /api/memory/* - Memory management endpoints
- /api/compliance/* - GDPR compliance endpoints
- /api/monitoring/* - Monitoring endpoints
- /health - Health check

Middleware:
- CORS handling
- JWT verification
- Rate limiting
- Security headers
- Request logging
- Error handling
- Response masking

3. AUTHENTICATION SERVICE
──────────────────────────
Technology: Supabase Auth + JWT

Endpoints:
- POST /api/auth/signup - Create account
- POST /api/auth/login - Login
- POST /api/auth/refresh - Refresh token
- POST /api/auth/password-change - Change password

Features:
- Email/password authentication
- JWT token generation
- Token refresh mechanism
- Password hashing (bcrypt)
- Session management

4. MEMORY SERVICE
──────────────────
Technology: Node.js + Supabase

Endpoints:
- POST /api/memory/capture - Capture new memory
- GET /api/memory/retrieve - Get relevant memories
- GET /api/memory/list - List all memories
- PUT /api/memory/:id - Update memory
- DELETE /api/memory/:id - Delete memory
- GET /api/memory/search - Search memories

Features:
- Memory capture from AI tools
- AI refinement (Qwen3)
- Vector embedding generation
- Semantic search
- Importance scoring
- Memory deduplication
- Tiering (hot/warm/cold)

5. AI REFINEMENT SERVICE
─────────────────────────
Technology: Qwen3 API

Process:
1. Receive raw memory text
2. Send to Qwen3 with extraction prompt
3. Parse structured JSON response
4. Merge with existing memories
5. Remove duplicates
6. Assign importance scores
7. Generate vector embedding

Output:
{
  "category": "architecture|preferences|stack|tasks|decisions",
  "content": "Structured memory text",
  "importance_score": 0.0-1.0,
  "tags": ["tag1", "tag2"],
  "timestamp": "ISO8601"
}

6. FILE SYNC SERVICE
─────────────────────
Technology: Node.js + chokidar

Watches:
- .cursorrules (Cursor)
- CLAUDE.md (Claude Code)
- .github/copilot-instructions.md (GitHub Copilot)
- .windsurfrules (Windsurf)

Features:
- File system watching
- Auto-update on memory changes
- Conflict detection
- Format conversion per tool
- Atomic writes

7. MONITORING SERVICE
──────────────────────
Technology: Node.js + Winston

Tracks:
- Request metrics
- Response times
- Error rates
- Database performance
- API usage
- System health

Endpoints:
- GET /api/monitoring/metrics - System metrics
- GET /api/monitoring/backup-health - Backup status
- GET /api/monitoring/backup-report - Backup report

8. SECURITY LAYER
──────────────────
Features:
- HTTPS enforcement
- JWT authentication
- AES-256 encryption at rest
- Rate limiting (100 req/15 min)
- Security headers (CORS, CSP, X-Frame-Options)
- Input validation
- SQL injection prevention
- XSS protection
- CSRF protection

9. DATABASE LAYER
──────────────────
Technology: Supabase PostgreSQL + pgvector

Tables:
- users: User accounts and auth
- memories: Memory storage with vectors
- projects: Project metadata
- audit_logs: Security audit trail

Features:
- Foreign key constraints
- Row-level security (RLS)
- Automatic timestamps
- Vector indexing
- Full-text search
- Automatic backups

═══════════════════════════════════════════════════════════════════════════════
SECURITY ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

AUTHENTICATION
───────────────
1. User signs up with email
2. Password hashed with bcrypt
3. JWT token generated
4. Token sent to client
5. Client includes token in requests
6. Backend verifies token
7. Token expires after 24 hours
8. Refresh token used to get new token

ENCRYPTION
───────────
1. Passwords: bcrypt hashing
2. Data at rest: AES-256 encryption
3. Data in transit: HTTPS/TLS
4. Sensitive fields: Masked in responses

AUTHORIZATION
───────────────
1. JWT token contains user ID
2. Backend verifies user owns resource
3. Row-level security (RLS) in database
4. Rate limiting prevents abuse
5. Input validation prevents injection
